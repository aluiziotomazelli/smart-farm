verificar se o ESP32 reiniciou após uma atualização OTA bem-sucedida usando o mecanismo de estado de imagem OTA do ESP-IDF.￼

Após chamar esp_https_ota() e esp_restart(), no próximo boot, o firmware recém-atualizado inicia no estado ESP_OTA_IMG_PENDING_VERIFY.￼ Esse estado indica que a aplicação é nova e precisa ser validada.￼

Para confirmar que a atualização foi bem-sucedida e evitar um rollback automático, seu código deve:

Verificar o estado da partição atual durante o boot.
Executar testes (diagnósticos) para garantir que o novo firmware está funcionando corretamente.
Chamar esp_ota_mark_app_valid_cancel_rollback() se tudo estiver OK.￼
Exemplo de código:
const esp_partition_t *running = esp_ota_get_running_partition();
esp_ota_img_states_t ota_state;

if (esp_ota_get_state_partition(running, &ota_state) == ESP_OK) {
    if (ota_state == ESP_OTA_IMG_PENDING_VERIFY) {
        bool diagnostic_ok = realizar_diagnostico(); // Sua função de teste
        if (diagnostic_ok) {
            esp_ota_mark_app_valid_cancel_rollback();
        } else {
            esp_ota_mark_app_invalid_rollback_and_reboot();
        }
    }
}
